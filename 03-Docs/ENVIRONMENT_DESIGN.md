## 1. 研究目标

研究提案提出三个核心问题：

1. **策略生态与主导**：在动作空间不受限的真实市场中，哪类策略会胜出？语言模型是否会产生人类策略库之外的新策略？
2. **价格形成**：在数百至上千轮交易中价格如何演化——收敛到竞争均衡（Smith, 1962）、呈现 Chamberlin 式的持续偏离、还是产生新的动态？
3. **人类策略与生成式策略的比较**：以真人偏好为基础的策略与语言模型自发生成的策略相比，哪一类福利更高？

提案的一条硬约束是：**不预先写入任何行为规则**——被研究的策略必须由智能体自行生成。

本环境采用该约束的最严格形式：**所有智能体在同一套公平规则下自行运行；研究者不指定任何智能体带有偏差，也不向任何子群体提供特殊或扭曲的信号。** 若市场出现可被利用的偏差，它必须由智能体自身产生，研究的任务是观察是否有智能体发现并利用了它。

## 2. 设计原则：可被利用的结构从何而来

一条决定整体设计的原则：

> **策略是对某种"可被利用的结构"的利用。环境中若不存在这种结构，就不会有有意义的策略涌现——智能体只会收敛到真实概率，留不下可供研究的对象。**

文献为此提供了一个佐证，同时是一个警告：当一组同质的理性语言模型相互竞争时，价格通常停在基本面价值附近，策略多样性很低（[arXiv 2502.15800]）。

在本环境中，研究者**不注入**这种结构。可被利用的结构只允许来自两个**非循环**的来源：

- **(A) 智能体之间的解读差异与水平差异。** 即便所有智能体获得相同且无偏的信息，它们对信息的理解、风险态度、推理深度仍会不同，由此产生分歧、错价以及随之而来的获利机会。
- **(B) 语言模型自身的偏差。** 语言模型在训练中可能习得类似人类的认知倾向（例如对耸动信息的过度反应）。这类偏差若出现，是被观察、被测量的现象，而非被植入的设定。

需要明确的一个边界：纯粹的信息结构只能产生噪声与信息差，**不必然**产生方向性的系统偏差（例如对小概率事件的系统高估）。方向性偏差本质上是偏好或认知现象，其非循环的来源只能是 (B) 语言模型自身，或以真人偏好为基础的策略（提案的 Arm 2）。

由此引出一个判据，用于区分"合法的环境设计"与"不当的预设"：

> 可以设计的是**被利用的环境**（信息、市场、制度、激励）；不能预设的是**被发现的策略本身**，也不能预先指定哪些智能体犯错。

这一判据与实验经济学的传统一致：Smith（1962）与 Chamberlin（1948）都为被试手动设定诱导价值（induced values），而交易行为本身自由；Gode 与 Sunder 的零智能（ZIC）模型是被广泛引用的手写基准。环境结构由研究者设定是标准做法，被研究的策略由智能体自由生成。

## 4. 目标环境结构（分层）

整体可理解为一个生态系统：研究的是哪类策略会自发涌现并胜出，因此环境需要同时提供产生分歧的条件、自由的动作空间、足够长的时间跨度，以及完整的测量手段。

```
L6  测量层    校准曲线 / 策略聚类 / 财富分布 / 新颖性检测
L5  时间层    T≥1000 轮 / 周期结算 / 财富滚动 / 滚动开市
L4  动作层    读取 + 下单/撤单 + transfer + create_market
L3  参与者层  待研究的语言模型 + 做市 + 套利对照，群体多样、信息对称
L2  制度层    连续双拍卖、价格-时间优先 / YES+NO 铸造合并 / 费用等
L1  市场层    多市场 / 低基率为主 / 公共耸动度属性 / 相关市场组
L0  信息层    隐状态、预采样结局、对所有人对称的新闻
```

### L0 信息层

信息层决定智能体形成信念的依据，也是涌现偏差能否出现的前提。本设计的原则是**对所有智能体对称且公平**：差异只允许来自各自的解读，而非来自研究者给予的不同信息。

- 每个市场背后有一个隐状态与一个预采样的真实结局（沿用现状 `simulation.py:111-117`，结算不消耗随机性）。
- 信息以两种公平形式之一提供：
  - **公共新闻**：所有智能体看到同一条关于隐状态的、带噪声且可能带歧义的消息，各自折算为概率。这是最纯粹的形式——分歧完全来自解读。
  - **对称私有信号**：所有智能体从同一个无偏分布抽取各自的带噪信号（即当前实现，没有任何群体被偏袒）。这种形式制造更多分歧与交易。
- 新闻**随机、稀疏地到达**（例如泊松到达），使市场有消化期，可形成错价又被纠正。当前私有信号每轮发放，应改为随机稀疏到达。

不在主设计中采用的做法（以保持纯涌现）：向部分智能体提供他人没有的信号（信息不对称），或直接告知部分智能体真实概率的偏移方向。信息不对称若要研究，应作为后期明确标注的受控对照（见 L2 制度变体），与纯涌现基线分开。

**拟改造配置（`NewsConfig`，`runner/config.py:31`）** ⬜：`arrival`（到达过程）、`rate`（到达强度）、`as_world_event`（是否发送需解读的世界新闻，否则回退为概率读数）。

### L1 市场层

- **多市场**（最小版本 K≥10，正式实验更多）。跨市场的策略（例如对一类事件系统性地反向下注）只有在市场群中才存在。
- **以低基率为主**：多数市场的真实 YES 概率偏低，贴近真实平台的分布；实证显示零售参与者倾向高估尾部事件（[Reichenbach & Walther]）。
- **公共耸动度属性**：为市场/新闻标注一个公开的"耸动度"，对所有智能体一视同仁。它本身不是偏差注入，而是一个公共属性，用于检验语言模型是否会对耸动框架产生过度反应。
- **相关市场组**：例如 `A`、`B`、`A∨B`，价格应满足自洽关系。三者不自洽时存在套利机会，可观察智能体是否发现。

**拟改造配置（`MarketConfig`，`runner/config.py:15`）** ⬜：`drama`（耸动度，0..1）、`group_id` 与 `group_relation`（相关市场组）。

### L2 制度层

- **连续双拍卖、价格-时间优先**（提案规格）；YES/NO 铸造/合并守恒、整数分定价——引擎核心已具备。
- 现有的盲提交回合清算保留为确定性/回放模式；连续双拍卖作为正式实验模式，以一个开关切换，不影响既有回放能力。
- 预留制度变体开关（提案的 institutional variation；`config.py:55-58` 已有部分占位）：费用、是否允许市价单、实名或匿名、是否允许开市、是否引入信息不对称等。这些是后期"制度 × 策略"对照实验的自变量。

**拟改造配置（`Config`）** ⬜：`matching`（`blind` 或 `cda`）、`disclose_identity`。

> 关于撮合规则与"事件驱动"：连续双拍卖比盲提交更接近实时撮合。但对策略涌现而言，瓶颈在于是否存在可被利用的结构（L0/L1/L3），而非撮合的微观规则。因此连续双拍卖的优先级低于信息层与参与者层。

### L3 参与者层

参与者群体决定了整个环境能否产生分歧与错价。设计要点是：群体不能同质，但异质性的来源必须是公平的多样性，而非研究者注入的偏差（详见第 5 节）。

| 角色 | 定位 | 实现 |
|---|---|---|
| 待研究的语言模型 | 研究对象。Arm 1 自由生成策略；Arm 2 以真人偏好为基础 | `ToolLoopAgent`（`llm_agent.py:282`）已有 |
| 做市者 | 提供流动性，赚取价差，可加入或退出 | `NaiveMM`（`scripted.py:49`）已有 |
| 套利对照（可选） | 机械地消除套利，作为基准与对照 | 新增或复用 `Fundamentalist` |

为使"一部分智能体犯错、另一部分发现并利用"这一结构有机会出现，群体需要内部多样性：不同模型、不同采样温度、不同的中性人设。这是公平的多样性，不向任何子群体注入偏差。

手写的偏差智能体（如带概率加权的零售模型）**不进入正式实验**。它仅在开发期用于**仪器校验**：植入一个已知偏差，确认市场确实可被利用、且测量管线（如校准曲线）能够检测到该偏差；校验完成即移除。其唯一例外用途是作为明确标注的受控自变量出现在专门的因果对照实验中（例如研究给定幅度的错价多久被纠正、由谁纠正），此时它是公开的处理变量而非隐藏假设。

> 平衡说明：套利对照与 `Fundamentalist` 会把价格推回真值、消除错价。研究"错价能持续多久"时，应限制此类力量的数量、资金或速度——错价的半衰期本身就是研究产出。

### L4 动作层

补齐提案明确列出的动作，使动作空间真正不受限：

- 读取：`get_markets / get_orderbook / get_trade_history / get_portfolio / get_news / get_news_detail`（✅ 已有）
- 交易：`place_order / cancel_order`（✅ 已有）
- `transfer(to, amount)` ⬜：使结盟、补偿、合谋等行为有可能涌现（文献显示合谋可自发出现，且连续观察是其形成与被报复的前提，[arXiv 2507.01413]）
- `create_market(question, resolution)` ⬜：使"自建市场"这类元策略有可能涌现，对应研究问题中"人类策略库之外的新策略"
- 自省钩子：`commit_view / finish`（✅ 已有），用于事后重建策略

**实现位置** ⬜：在 `agents/llm_agent.py` 的 `_build_agentic_tools` 增加两个函数声明，并在 `runner/simulation.py` 增加对应的引擎动作与事件类型。

### L5 时间与动态层

对齐提案的仿真协议：

- 运行 **T≥1000 轮**；
- **每约 100 轮周期性结算**、派付、并把财富滚动到下一阶段；
- **滚动开设新市场**（可包含智能体自建市场）；
- 财富滚动与路径依赖，是观察财富集中、策略存活与淘汰的前提；短程单次结算无法呈现演化。

**拟改造配置** ⬜：`resolution_every`（周期结算间隔）、`rolling_markets` 与 `new_markets_per_cycle`。

### L6 测量层

环境不仅要能运行，还要能让涌现被观察到。事件溯源日志（`runner/sinks.py`）已具备，需补充以下产出 ⬜：

- **校准曲线**：市场隐含概率与实际结局的对比，用于判断是否出现内生错价、以及是否被利用。
- **策略分类**：按行为特征（撤单频率、反向下注比例、价差形态、持仓集中度）聚类，建立涌现策略的分类体系。
- **财富动态**：基尼系数、财富流动性、策略类型的存活分析。
- **新颖性检测**：识别在真人策略库（Arm 2）中没有对应物的语言模型策略。

可复用现有评测框架的产出管线（`eval/metrics.py`、`eval/scorecard.py`）。

## 5. 一个关键区分：群体多样性与偏差注入

纯涌现设计拒绝向子群体注入偏差，但仍需要一个公平且有分歧的擂台。下表区分两者，以免把必需的公平多样性与不当的偏差注入混为一谈。

| 公平、必需（不属于偏差注入） | 应当避免（属于偏差注入） |
|---|---|
| 群体多样性：不同模型、温度、中性人设，制造观点分歧 | 给某子群体"非理性散户"之类的偏差人设 |
| 同一条带歧义/耸动的新闻发给所有智能体 | 只向部分智能体提供某一方向的信号 |
| 事件本身较难、低基率，留出犯错空间 | 直接告知部分智能体真实概率的偏移 |
| 长程与多市场，使小幅优势累积到可测 | 用固定公式手写一批偏差智能体并纳入正式实验 |

核心：所有智能体获得相同的信息，差异只来自各自的解读与水平。耸动偏差若出现，是语言模型在解读耸动新闻时自行产生的，属于纯涌现现象。

一个现实约束：若所有智能体使用同一模型与同一提示，其行为会高度一致，难以形成"一部分犯错、一部分套利"的结构。因此群体多样性（多模型、多温度、多中性人设）是使涌现有机会发生的必要条件，而它本身不构成偏差注入。

## 6. 最小可运行版本

一个尽量小、但仍能支撑研究问题的配置：

- 约 10 个市场，以低基率为主，其中若干标注耸动度，并含一组 3 个相关市场（`A`/`B`/`A∨B`）；
- 信息对所有智能体对称：同一条需解读的世界新闻，随机稀疏到达；
- 群体由多样化的待研究语言模型（Arm 1）加 1–2 个做市者组成；信息差异仅来自解读；
- 动作集补齐 transfer 与 create_market；
- 撮合先用现有盲提交亦可（连续双拍卖后置），运行约 1000 轮、每 100 轮结算、财富滚动；
- 产出校准曲线、盈亏分布与策略聚类。

在此配置下，反向下注、跨市场套利、做市、动量等策略都有存在空间但均未被预先写入；它们能否自发出现、哪一类胜出、以及是否出现新策略，正对应三个研究问题。手写偏差智能体仅在此版本的开发期用于校验测量管线，不进入正式运行。

## 8. 设计决策与预先登记的结果

**主设计为纯涌现（Arm 1）+ 公平信息。** 研究者不注入偏差、不偏袒任何子群体。可被利用的结构只允许来自智能体的解读差异与语言模型自身偏差。

**两种结果都预先登记为合法结论：**

1. **出现涌现**：语言模型自发产生方向性偏差（如对耸动事件系统高估 YES），且有智能体学会反向利用并获利。这是对研究问题的正面回答。
2. **未出现涌现（null result）**：语言模型在相互竞争中收敛到真实概率，错价不持续、无可利用优势。文献已提示这种可能（[arXiv 2502.15800]）。结论"此类语言模型市场过于有效，不复现人类市场异象"本身具有价值。

**方向性偏差的来源限制**：纯信息结构不足以产生方向性系统偏差，其非循环来源只能是语言模型自身（Arm 1）或以真人偏好为基础的策略（Arm 2）。若 Arm 1 呈现 null result，则反向下注一类的研究主要由 Arm 2 承载。

**待确认的决策**：

- 信息层采用公共新闻还是对称私有信号，或两者作为对照；
- 是否、以及何时将信息不对称作为后期受控制度变体引入；
- 群体多样性的具体构成（模型种类、温度区间、中性人设集合）；
- 概率加权等仪器校验工具的参数取值（仅用于校验，不进入正式实验）。
